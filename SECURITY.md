# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in AIMap, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, please email **security@bishopfox.com** with:

- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will acknowledge receipt within 48 hours and provide a detailed response within 5 business days.

## Supported Versions

| Version | Supported |
|---------|-----------|
| latest  | Yes       |

## Scope

This policy applies to vulnerabilities in the AIMap codebase itself. Issues related to third-party dependencies should be reported to the respective maintainers, though we appreciate a heads-up if they affect AIMap users.
